About the Project

PyPass is a command-line password manager that stores all passwords locally on your own computer, fully encrypted. There is no cloud, no server, and no third party involved. Even if someone gets access to the vault file, they cannot read anything without the master password.

The main idea came from a simple observation: most people use cloud-based password managers, which means their passwords are stored on someone else's server. PyPass solves this by keeping everything offline and encrypted on the user's own machine.

Research Question

How can AES-256 encryption and PBKDF2 key derivation be used to protect a local password database, and what are the main security benefits of keeping this data offline?

How It Works

  1. User sets a master password on first run. This password is never stored anywhere.
  2. A random 32-byte salt is generated and saved in the vault file.
  3. The master password + salt go through PBKDF2-HMAC-SHA256 (310,000 iterations) to produce an AES-256 key.
  4. Each password entry is encrypted with AES-256-CBC using a random IV.
  5. An HMAC-SHA256 is added to detect any tampering with the vault file.
  6. Everything is saved locally as a JSON file. No internet connection needed.

Security Features

  • AES-256-CBC — industry standard symmetric encryption. The same algorithm used by banks and governments.
  • PBKDF2-HMAC-SHA256 — key stretching with 310,000 iterations makes brute force attacks extremely slow.
  • Random salt — a unique 32-byte salt means the same master password produces a different key every time.
  • Random IV — a new 16-byte IV for every encryption means the same password encrypts differently each time.
  • HMAC-SHA256 — integrity check detects if the vault file was modified or corrupted.
  • Local only — no cloud, no network, no third party. Data never leaves the user's computer.
