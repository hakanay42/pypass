PyPass - Password Manager

What is the problem?

Most people use cloud-based password managers. The problem is your passwords are stored on someone else's server. If that server gets hacked, your passwords are exposed. My project stores everything locally on your own computer, fully encrypted.

Research Question

How can AES-256 encryption and PBKDF2 key derivation be used to protect a local password database, and what are the security benefits of keeping data offline?

How it works

  • User creates a master password on first run
  • The master password is used to generate an AES-256 key using PBKDF2
  • All passwords are encrypted and saved in a local file
  • Without the master password, the file cannot be read

Encryption used

  • AES-256-CBC - encrypts the passwords
  • PBKDF2-HMAC-SHA256 - creates the key from master password (310,000 iterations)
  • Random salt - makes sure same password gives different key each time
  • HMAC - detects if the file was changed or corrupted

How to run

pip install cryptography
python pypass.py

What this prototype can do

  • Create a new encrypted vault with a master password
  • Add a password for a service
  • View a saved password
  • List all saved services

What the final version will have

  • Delete passwords
  • Generate random strong passwords
  • Change master password
